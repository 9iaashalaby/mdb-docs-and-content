Deshabilitar el ajuste de texto

Si no desea que el texto del botón se ajuste, puede agregar el .text-nowrap al botón. En Sass, puedes configurar $ btn-white-space: nowrap para deshabilitar el ajuste de texto para cada botón.

Colores

MDB incluye varios estilos de botones predefinidos, cada uno con su propio propósito semántico, con un pocos extras para un mayor control.

Transmitir significado a las tecnologías de asistencia:
El uso del color para agregar significado solo proporciona una indicación visual, que no se transmitirá a usuarios de tecnologías de asistencia, como lectores de pantalla. Asegúrese de que la información indicada por el color es obvio por el contenido mismo (por ejemplo, el texto visible), o está incluido a través de medios alternativos, como texto adicional oculto con el .sr solo clase.

contorno

¿Necesitas un botón, pero no los colores de fondo que traen? Reemplazar el predeterminado clases modificadoras con las .btn-outline- * para eliminar todas las imágenes de fondo y colores en cualquier botón.

En los botones de esquema, recomendamos agregar data-ripple-color = "dark" para cambiar el color del efecto dominó . El color de luz predeterminado de la ondulación (aplicado automáticamente a cada botón) puede no ser bien visible en el caso de los botones de luz y contorno.

Para obtener más información sobre el efecto dominó y todas las opciones disponibles, consulte Documentos de Ripple .

Redondeado

Añadir .btn-redondeado class para redondear el botón.

Contorno redondeado

Puede usar .btn-outline- * y .btn-round juntos para hacer contorno del botón y redondeado al mismo tiempo.

Flotante

Utilice la clase .btn-floating para hacer un botón circular.

Para que funcione correctamente hay que poner un icono en su interior. El texto no encajar. Puede encontrar cientos de iconos disponibles en nuestro documentos de iconos .

Puede aplicar casi todas las mismas clases y atributos a los botones flotantes que al botones regulares: colores, ondas, tamaños, contorno, etc.

Etiquetas

los .btn Las clases están diseñadas para ser utilizadas con <botón> elemento. Sin embargo, también puede utilizar estas clases en <un> o <entrada> elementos (aunque algunos navegadores pueden aplicar una representación ligeramente diferente).

Al usar clases de botones en <a> elementos que se utilizan para activar in-page funcionalidad (como contraer contenido), en lugar de vincular a nuevas páginas o secciones dentro la página actual, estos enlaces deben tener unrole="button" apropiadamente transmitir su propósito a las tecnologías de asistencia, como los lectores de pantalla.

Estado discapacitado

Haga que los botones parezcan inactivos agregando el disabled atributo booleano a cualquier <button> elemento. Los botones deshabilitados tienen pointer-events: none aplicado, evitando que los estados activos y de desplazamiento desencadenante.

Botones desactivados con el <a> el elemento se comporta un poco diferente:

  • <a>s no apoye el disabled atributo, por lo que debe agregar la .disabled class para que parezca visualmente deshabilitado.
  • Se incluyen algunos estilos compatibles con el futuro para deshabilitar todos pointer-events en los botones de anclaje. En los navegadores que admiten esa propiedad, no verá el cursor inhabilitado en absoluto.
  • Los botones deshabilitados deben incluir el aria-disabled="true" atributo para indicar el estado del elemento a tecnologías de asistencia.
