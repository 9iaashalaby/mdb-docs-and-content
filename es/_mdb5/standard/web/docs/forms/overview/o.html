Cada grupo de campos de formulario debe residir en un <form> elemento. MDB proporciona sin estilo predeterminado para el <form> elemento, pero hay algunos poderosos características del navegador que se proporcionan de forma predeterminada.

  • ¿Es nuevo en los formularios del navegador? Considere revisar los documentos del formulario MDN para obtener una descripción general y una lista completa de los atributos disponibles.
  • <button>s dentro de un <form>predeterminado a type = "submit" , así que esfuércese por ser específico y siempre incluya un tipo .
  • Puede deshabilitar cada elemento de formulario dentro de un formulario con el disabled atributo en el<form>.

Dado que MDB aplica display: block y width: 100% a casi todos nuestros controles de formulario, los formularios se apilarán verticalmente por defecto. Se pueden usar clases adicionales para variar este diseño en función de cada formulario.

Diseño

Hay varias formas de estructurar formularios y proporcionarles el diseño deseado. Mira esto los ejemplos siguientes para obtener más información sobre el diseño de formularios.

Utilidades

Las utilidades de margen son la forma más sencilla de agregar algo de estructura a las formas. Proporcionan agrupación básica de etiquetas, controles, formulario opcional mensajes de texto y validación de formularios. Recomendamos ceñirse a utilidades margin-bottom y utilizando una sola dirección en todo el formulario para consistencia.

Siéntase libre de crear sus formularios como desee, con <fieldset>s, <div>s, o casi cualquier otro elemento.

Cuadrícula de formulario

Se pueden construir formas más complejas usando nuestro clases de cuadrícula . Úselos para diseños de formularios que requieren múltiples columnas, anchos variados y opciones de alineación adicionales. Requiere que la variable $ enable-grid-classes Sass esté habilitada (activado de forma predeterminada).

Canalones

Al agregar clases de modificador de canaleta, puede tener control sobre el ancho de la canaleta también en en línea como dirección del bloque. También requiere que la variable $ enable-grid-classes Sass esté habilitada (activado de forma predeterminada).

Column sizing

Como se muestra en los ejemplos anteriores, nuestro sistema de cuadrícula le permite colocar cualquier número de .col s dentro de un .row . Dividirán el ancho disponible por igual entre ellos. También puede elegir un subconjunto de sus columnas para ocupar más o menos espacio, mientras que los .col restantes dividen por igual el resto, con clases de columna específicas como .col-sm-7 .
